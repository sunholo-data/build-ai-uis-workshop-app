This sprint's "smoke" is the integration test in M2, not a chrome
screenshot. Tenant attribution is a backend-only gate with no
user-visible UI surface — every span emitted during a request
carries the right attrs (or doesn't). The integration test asserts
that exact property using OTel's InMemorySpanExporter, which is
the canonical pattern for testing SpanProcessors.

What the tests prove end-to-end:

1. backend/tests/unit/test_tenant_context.py — 19 unit tests (M1: 15, M3: +4)
   M1 covers:
   - Contextvar set/get with all four PII-safe defaults
   - tenant.uid + tenant.auth_mode always present
   - tenant.group_id only when non-empty
   - tenant.uid_hash only when email present
   - tenant.uid_hash is 64-char hex SHA256 digest, NOT the raw email
   - tenant.uid_hash is deterministic (same email → same hash)
   - SpanProcessor stamps every attr onto the span
   - SpanProcessor with no context emits unchanged (no crash)
   - Enricher exception swallowed + WARN log + others still run
   - register_tenant_enricher rejects non-callable (fail-loud)
   - Multiple enrichers compose with last-wins on key collision
   - clear_tenant_enrichers() resets the registry
   - **Concurrent-task isolation via asyncio.gather + barriers**
     (the headline correctness test — proves zero cross-tenant leakage)

   M3 hardening adds:
   - Golden SHA256 verification — pins the hash function against
     accidental downgrade to MD5 / truncated / custom hash
   - UserWithDisplayName duck-type test — proves the impl reads
     fields explicitly (no __dict__ walk); future User subclass
     with display_name cannot leak through this code path
   - Scan-every-value defence-in-depth — no raw email under ANY
     key, no key contains 'email'
   - extra-kwarg override documented (explicit, intended for tests)

2. backend/tests/integration/test_tenant_attribution.py — 5 integration tests
   - Firebase path: tenant.uid + tenant.auth_mode='firebase' +
     tenant.uid_hash (SHA256) — raw email NEVER on the span
   - Group-auth path: tenant.group_id stamped from User.group_id;
     no uid_hash since anon-group users have no email
   - LOCAL_MODE stub path: tenant.auth_mode='local_mode_stub' +
     stub uid
   - Concurrent-tenant isolation via asyncio.gather + barriers
     (the FULL pipeline including get_current_user dispatcher,
     not just the contextvar — proves zero cross-leakage end-to-end)
   - Direct dispatcher regression — get_current_user calls
     set_tenant_context(user) before returning, regardless of which
     auth path resolved the User

End-to-end coverage:
  request →
  get_current_user dispatches to one of three auth paths →
  _resolve_user returns User →
  set_tenant_context(user) writes platform defaults +
    runs every registered enricher →
  contextvar bound on the current async task →
  endpoint handler + ADK runner + tools + LLM calls all run in
    the same task →
  TenantAttributeSpanProcessor.on_start reads contextvar +
    stamps attrs onto every started span →
  exporter sees spans carrying tenant.uid + tenant.auth_mode +
    optionally tenant.group_id + optionally tenant.uid_hash

Each link in the chain has dedicated tests. The integration tests
in test_tenant_attribution.py are the single artifact that prove
the full chain end-to-end without mocking the boundary between
contextvar + SpanProcessor (the exporter sees real spans).
